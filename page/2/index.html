关于路由模块的相关知识

关于路由模块的相关知识

  • 由于用Node启动的文件(入口文件)处理所有请求,有弊端:
    1. 代码太多
    2. 不利于多人协作开发

  • 解决:

    使用路由+模块分别来处理我们的请求

    【注意】: 模块职责要单一

  • Express提供了一个API Router()专门用来包装路由的

    (1) 路由模块的职责:

    处理路由

    根据不同的请求方式+模块设置具体的请求处理函数

    router.js步骤:

  1. 导入express模块
    const express = require(‘express’)
  2. 创建一个路由容器
    const router = express.Router()
  3. 把路由都挂载到路由容器中
    router.get(…)
    router.post(…)
  4. 通过module.exports的方法把路由容器导出
    module.exports = router

(2) 入口模块app.js的职责:

  1. 创建服务
  2. 做一些服务相关的配置
    例: ① 模板引擎
    ② bod-parser解析表单post请求体
③ 提供静态资源服务
  3. 绑定端口启动服务

自定义模块的相关知识

自定义模块的相关知识

一、自定义模块且自己使用:

  1. 导出:

    module.exports

    • 直接导出(不建议):

    一个文件/模块只能导出一个
    module.exports = 成员

    • 以对象的形式导出(建议使用):

    导出对象【三种写法】

    ① module.exports = {}
    ② module.exports.属性1 = 值1
    module.exports.属性2 = 值2

    ③ exports.属性1 = 值1
    exports.属性2 = 值2

    注意:不能这么写 exports = {}

  1. 导入 require()

    【注意】

    核心模块和第三方模块导入直接写名就行
    自定义模块导入要写全路径 用path

二、 将自定义模块上传到 npmjs.org

前提:
​

  • ① 注册账号并且使用邮箱激活
  • ② 项目中，必须要有package.json

name:别人导入你的时候，要写的名称
main:加载时候，指向的这个文件,如果没有指定就找 到当前第三方包根目录的index.js
keywords:搜索关键字,让别人能搜索你

可以直接 在根目录打开黑窗，npm init -y 自动生成package.json,注意要去修改或填写 name,main,keyword…

  • ③ 使用 npm 这个工具上传【都是要在项目根目录】

1、npm adduser【连接到npmjs.org】
2、npm publish 发布即可
​
注意:

(1) 包的名称必须唯一
(2) 如果更改了包之后，想再次发布，必须版本号递增
(3) 删除某个版本: npm unpublish xncalc@1.0.0

nodejs URL模块操作URL的parse方法

url.parse方法总结

url.parse(urlString[, parseQueryString[, slashesDenoteHost]])

urlString: 获取的url字符串
parseQueryString: 布尔类型

false: 默认; 返回的是URL对象,且URL对象中的query属性是未解析的字符串
true: 返回的是URL对象,且URL对象中的query属性也是对象

slashesDenoteHost:布尔类型

false: 默认;
true: 能够解析不带协议的url请求

css3背景设置

背景

css3中背景的新特性介绍

  • c3中增强了背景的设置，可以设置背景的以下内容
    • background-size: 背景的大小
    • background-position: 背景起始显示的位置(x和y方向)
    • 一个盒子可以设多个背景图片

背景size设置

  • 比如盒子是100 100. 背景图片是 200200. 可以通过设置背景图片的大小，来适应盒子
  • background-size
    • auto： 背景图像的真实大小。
    • cover： 背景图片等比例缩放到铺满盒子
    • contain： 背景图片拉伸以适应盒子大小
    • 直接设置大小 background-size: 100px 100px; 或者backgound-size: 10% 10%;

背景位置的设置

  • background-position: 背景起始显示的位置(x和y方向),可以有以下几种写法
    * left top, left bottom, right top, right bottom
* 50% 50%
* 10px 10px;
  • 如果同时设置position和size，复合写法需要用 postion/size 的格式
  • 复合写法： background: url(img/box-sizing.png) left top/100px 100px no-repeat

多张背景的设置

  • 多张背景图片设置之间用逗号隔开

css3渐变

渐变

介绍

  • CSS3 渐变（gradients）可以让你在两个或多个指定的颜色之间显示平稳的颜色过渡。
  • 通过使用 CSS3 渐变（gradients）
  • 渐变（gradient）是由浏览器生成的， 以前实现类型的效果要使用图片
  • CSS3 定义了两种类型的渐变（gradients）：
    • 线性渐变（Linear Gradients）- 向下/向上/向左/向右/对角方向
    • 径向渐变（Radial Gradients）- 由它们的中心定义
  • 要想让渐变生效，需要把渐变设给background

线性渐变(linear)

  1. 默认是从上往下渐变

  2. 自定义渐变方向

  3. 重复渐变

径向渐变

  • 径向渐变是一个圆形的渐变，由圆心向四周发散的渐变
  • 需要设置的参数包括：
    • 颜色（可以设多个）
    • 中心 (at center center)
      • 语法（at x位置 y位置) 都是从左上角原点为参考
      • x, y可以是像素 (at 30px 50px)
      • 可以是百分比 (at 10% 30%)
      • 也可以是top， left， right，center (at left bottom)
      • at center center (默认值， 圆心在盒子的正中心位置)
    • 大小
      • closest-side（最近边）
      • farthest-side（最远边）
      • closest-corner（最近角）
      • farthest-corner（最远角, 默认值）
    • 形状（默认是椭圆
      • elllipse（椭圆，默认值）
      • circle
