关于路由模块的相关知识

  • 由于用Node启动的文件(入口文件)处理所有请求,有弊端:
    1. 代码太多
    2. 不利于多人协作开发

  • 解决:

    使用路由+模块分别来处理我们的请求

    【注意】: 模块职责要单一

  • Express提供了一个API Router()专门用来包装路由的

    (1) 路由模块的职责:

    处理路由

    根据不同的请求方式+模块设置具体的请求处理函数

    router.js步骤:

  1. 导入express模块
    const express = require(‘express’)
  2. 创建一个路由容器
    const router = express.Router()
  3. 把路由都挂载到路由容器中
    router.get(…)
    router.post(…)
  4. 通过module.exports的方法把路由容器导出
    module.exports = router

(2) 入口模块app.js的职责:

  1. 创建服务
  2. 做一些服务相关的配置
    例: ① 模板引擎
    ② bod-parser解析表单post请求体
③ 提供静态资源服务
  3. 绑定端口启动服务
