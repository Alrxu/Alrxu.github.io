使用vue-cli脚手架生成项目以及项目分析

使用脚手架生成Vue项目

vue-cli脚手架的作用
    1、帮助我们生成项目
    2、开发阶段会生成开启一个Web服务器来运行我们前端的代码，
    能够看到写的代码的效果
    3、生产阶段，对我们写的代码进行压缩与优化

生成项目

前提:
    npm install -g @vue/cli

生成项目
    vue create my-project

分析项目的组成

szhmqd23_vue
    |--node_modules 第三方包安装目录
    |--public 项目打包之后的目录
    |--src 项目源代码目录
        |--main.js 打包的入口文件
        |--App.vue 根组件
    |--.gitignore git的忽略文件
    |--.prettierrc 格式代码的配置文件
    |--babel.config.js babel的配置文件，用来进行es6转es5
    |--package.json 项目的配置文件

分析项目运行起来的第一个组件

npm run serve ---> 
webpack去打包(打包src/main.js文件) --->
启动Node的服务，去监听8080端口，运行我们打包之后的代码（是通过运行 public/index.html）去运行的

因为在package.json文件中配置了,所以 直接运行 npm run serve 就可以运行项目

更改项目的webpack配置

参考:https://cli.vuejs.org/zh/config/#%E5%85%A8%E5%B1%80-cli-%E9%85%8D%E7%BD%AE

​
node_modules 依赖文件包
public 项目打包完毕之后,生成的文件所在的目录
– index.html
src 源代码目录
– assets 图片资源
– component 其他组件
– filter 过滤文件
– router 路由文件
– App.vue 根组件
– main.js 入口文件
– statics 静态资源文件
– site
– css
– image
– jquery_plugins jquery插件文件

  • public中的index.html由入口文件中的vue实例控制

  • 在入口文件main.js中

    • 导入所有需要打包的文件

    • 创建实例对象

      main.js入口文件中,创建vue实例,要把根组件渲染到index.html中,只能用render

      而用render会把el所指定的容器中所有内容都覆盖
      所以在实例中 用render函数渲染根组件到容器中,也就是说根组件的页面内容完全覆盖了index.html中的内容
      所以index.html中写代码没用会被覆盖,所以在根组件中写整体结构

  • App.vue是根组件,整个项目从在这里开始,整体结构布局

    存放每个页面公有的头部,导航,底部
header
nav
router-view路由占位符,由路由分配组件延伸到其他页面,每个页面都是一个子组件
footer

    每个组件的结构: template script style

    template: 模板代码
    script: 用 export default导出组件对象,组件中需要的包直接在该组件中导入
    style: 组件中的style 写成 scoped表示是私有css样式

  • 路由: 

    路由组件导航 <router-link to=''>
路由占位符 <router-view>
定义组件
创建路由对象,规则:每个路由映射一个组件
在vm实例中挂载路由对象

  • 将路由步骤抽离:

    (1) 在App.vue根组件中

    -- 路由组件导航 <router-link to=''>
-- 路由占位符 <router-view>

    (2) 新建一个component组件文件夹,存放子组件
    (3) 新建一个router文件夹

    -- 导入vue-router和它所依赖的vue,以及导入需要分配路由的子组件
-- 创建路由对象,制定规则路由映射组件 (就需要子组件)
-- 把创建的路由对象暴露出去

    (4) 在App.vue组件中的vm实例中挂载路由对象

    在子组件中 template style script
    在script中
    – 用 export default导出组件对象
    – 子组件对象中有该页面的数据和方法
    – 子组件中需要的包直接在该组件中导入

    【注意】
    ​ 导入文件时，如果下面的代码需要用到包中的方法

    ​ 就用 import A from ‘包名’导入包，需要调用那个包的方法时,就直接 A.方法

例如  import $ from 'jquery'
     import axios from 'axios'

如果不需要就直接 import ‘路径’ 导包
